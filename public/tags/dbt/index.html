Dbt

Olist Data Engineering Project

🛒 Visão Geral Este projeto implementa um pipeline de dados ponta a ponta utilizando o dataset público da Olist. O objetivo é simular um ambiente real de engenharia de dados, transformando dados brutos em insights estratégicos. 🏗️ Arquitetura do Projeto A solução foi desenhada para ser escalável e reprodutível, utilizando as seguintes etapas: Extração e Carga (EL): O Apache Airflow orquestra a captura dos arquivos CSV e realiza a ingestão no banco de dados PostgreSQL. Transformação (T): Utilização do dbt para criar camadas de dados (staging e marts), aplicando regras de negócio e garantindo a qualidade dos dados. Conteinerização: Todo o ambiente (Airflow, Postgres) é gerenciado via Docker Compose, facilitando o deploy e a configuração. Visualização: Criação de Dashboards no Power BI consumindo as tabelas finais modeladas no dbt. 🚀 Principais diferenciais técnicos: Orquestração: Uso de DAGs no Airflow para gerenciar dependências. Modelagem SQL: Transformações complexas e criação de tabelas fato/dimensão via dbt. Infraestrutura como Código: Setup rápido de ambiente com Docker. 🔗 Confira o código completo no GitHub
